接下来我们了解抽象机器模型中的 Grid 和 Block 组织方式。

cuTile 内核由一个网格，也就是 Grid 中的多个逻辑线程块 Block 执行。

首先看一维网格。在一维网格中，Block 按照线性方式排列，从 Block 零、Block 一、Block 二，一直到 Block n。这种方式适合处理一维数组数据。

再看二维网格。Block 按照矩阵方式组织，有行和列的概念。比如 Block 零零表示第一行第一列，Block 一二表示第二行第三列。这种方式非常适合处理图像和矩阵数据。

Grid 还可以是三维的，就是多层二维网格堆叠在一起，适合处理体数据或三维张量。

让我们看右边的代码示例。启动一维网格时，使用 grid 等于 8,1,1 表示 8 个 Block 在一维上排列。启动二维网格时，使用 grid 等于 16,16,1 表示 16 乘 16 的网格。三维网格则是 8,8,8。

在内核函数中，我们使用 ct.bid 函数来获取当前 Block 的索引。ct.bid(0) 获取第一维的索引，ct.bid(1) 获取第二维的索引。

记住三个关键要点：第一，Grid 定义了 Block 的组织方式，可以是一维、二维或三维。第二，所有 Block 并行执行相同的内核代码。第三，使用 ct.bid 函数获取当前 Block 的索引位置。
