学习了理论和代码之后，让我们进入实践环节。

首先看左侧，"运行测试"。

将代码保存为 vector_add.py，在终端运行：python3 vector_add.py

如果一切正常，你会看到绿色的成功提示："✓ vector_add_example passed!"。这说明你的第一个 cuTile 程序成功运行了！

右侧是"性能分析"。

cuTile 程序可以像传统 CUDA 内核一样使用 Nsight Compute 进行性能分析。运行命令：ncu -o profile --set detailed python3 vector_add.py

这会生成详细的性能报告，包括内核执行时间、内存带宽利用率、计算单元利用率和并行度分析。通过这些指标，你可以找到性能瓶颈并进行优化。

最后，我给大家三条"最佳实践建议"。

第一，"Tile 大小选择"。通常选择 2 的幂次方，比如 16、32、64 或 128。要考虑 GPU 的共享内存大小，并通过实验找到最优值。

第二，"内存访问优化"。确保内存访问是连续的，合理使用 Tile 缓存，避免频繁的小数据传输，利用 GPU 的内存合并访问特性。

第三，"调试与验证"。先用小数据集测试，对比 CPU 计算结果确保正确性，使用 Nsight 工具进行分析，然后逐步增加问题规模。

记住这三点，你的 cuTile 程序会更加高效可靠。
