理解了 Tile 索引机制后，让我们看看一个完整的 cuTile 程序是如何从头到尾执行的。

"阶段一：数据准备"

首先定义向量大小为 4096 个元素，每个 Tile 包含 16 个元素。然后使用 CuPy 在 GPU 上创建输入数组 a 和 b，以及输出数组 c。

"阶段二：网格配置"

这一步很关键。我们需要计算需要多少个处理器。使用 cdiv 函数，也就是向上取整除法：4096 除以 16 等于 256。所以 grid 配置为 (256, 1, 1)，意味着启动 256 个处理器。

"阶段三：启动内核"

调用 ct.launch 函数，传入当前 CUDA 流、网格配置、内核函数 vector_add，以及参数元组。这一行代码会将任务提交到 GPU。

"阶段四：GPU 并行执行"

此时，256 个处理器在 GPU 上同时开始工作！每个处理器独立执行：加载自己的 Tile 数据，进行计算，然后存储结果。这就是真正的并行计算，256 个任务同时进行！

"阶段五：结果验证"

GPU 计算完成后，我们将结果数组从 GPU 传回 CPU，与预期结果进行对比，验证计算的正确性。

从数据准备到结果验证，整个流程清晰明了。cuTile 帮我们处理了复杂的底层细节，让我们可以专注于算法逻辑。
