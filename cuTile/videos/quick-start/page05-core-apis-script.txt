【第5页：核心 API - 四大基本操作 - 讲解旁白】

时长：约 70 秒

理解了 Tile 和 Kernel 的概念后，现在让我们学习 cuTile 的四大核心 API。

【停顿 1 秒】

第一个，"ct.load"。这个函数用于从 GPU 内存加载数据到 Tile。

它接收三个主要参数：array 是源数组，index 是 Tile 索引（注意，这不是元素索引，而是第几个数据块），shape 是 Tile 的形状，也就是每个数据块的大小。

【停顿 1 秒】

第二个，"ct.store"。这个函数将 Tile 数据存储回 GPU 内存。

参数包括：array 是目标数组，index 是要存储到的 Tile 索引位置，tile 是要存储的 Tile 数据。

【停顿 1 秒】

第三个，"ct.bid"。这个函数获取当前处理器的索引，也就是 Block ID。

它接收一个 dim 参数，表示维度：0 代表 x 维度，1 代表 y，2 代表 z。返回值是当前块的索引号，用于确定当前处理器应该处理哪个 Tile。

【停顿 1 秒】

第四个，"ct.launch"。这是启动 GPU 内核函数的关键 API。

它的参数包括：stream 是 CUDA 流对象，grid 是网格大小（决定启动多少个处理器），kernel 是要执行的内核函数，args 是传递给内核的参数元组。

【停顿 1 秒】

这四个 API 构成了 cuTile 编程的核心工具箱：bid 确定位置，load 读取数据，中间进行计算，store 保存结果，最后用 launch 启动整个流程。

简单、清晰、强大！
