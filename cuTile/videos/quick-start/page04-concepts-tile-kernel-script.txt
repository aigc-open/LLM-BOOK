【第4页：核心概念 - Tile 与 Kernel - 讲解旁白】

================================================
时长：约 60 秒

现在，让我们深入了解 cuTile 的两个核心概念：Tile 和 Kernel。

【停顿 1 秒】

首先看左侧，"概念一：Tile，数据块"。

Tile 是 cuTile 中的基本数据单元，它代表一块连续的、固定形状的数据，类似于 NumPy 数组的一个切片。

【停顿 0.5 秒】

让我们通过示意图理解。假设我们有一个存储在 GPU 全局内存中的完整数组，cuTile 会将它划分为多个 Tile。比如这里的 Tile 0、Tile 1、Tile 2、Tile 3。

【停顿 0.5 秒】

每个 Tile 都会被分配给一个处理器进行并行处理。处理器 0 处理 Tile 0，处理器 1 处理 Tile 1，以此类推。这就是 Tile 级别的并行计算。

【停顿 1 秒】

再看右侧，"概念二：Kernel，内核函数"。

Kernel 是在 GPU 上执行的计算函数。我们使用 @ct.kernel 装饰器来定义它，这个函数会在多个处理器上并行执行。

【停顿 0.5 秒】

来看代码示例。我们用 @ct.kernel 装饰器定义一个内核函数 my_kernel。在函数内部，首先通过 ct.bid(0) 获取当前处理器的索引，然后使用这个索引来加载对应的 Tile 数据，进行计算，最后存储结果。

【停顿 1 秒】

简单来说：Tile 是"数据单元"，Kernel 是"处理函数"。两者配合，实现高效的并行计算。
